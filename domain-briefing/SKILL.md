# Domain Briefing Skill — Faye专用领域知识提炼系统

## 核心逻辑

这个skill解决一个具体问题：**Faye经常需要和某个陌生专业领域的顶级专家深度交流，但没有时间从零建立认知。**

工作流分两个阶段，可以独立使用，也可以在同一次对话中串联：

```
阶段一（见面前）：extended search → 领域认知地图
      ↓
阶段二（见面后）：录音稿 + 指令 → 提炼专业知识 → 合并输出完整文档(.docx)
```

最终产出不是信息整理，而是：**一份让不懂这个领域的人能快速建立结构化认知、并看清对方机构/人物运作逻辑的完整文档。**

---

## PHASE 1：领域背调

### 触发条件

用户提到：
- 要见某个特定领域的专家/机构负责人
- 需要了解某个陌生专业领域
- 为某类合作做准备（该合作涉及非Faye专长的领域）

### Step 1：识别调研维度

从用户输入中提取：
- **核心领域**（例：知识产权/生命科学/量子计算）
- **具体机构/人物**（如有）
- **调研深度需求**（泛了解 vs 深度准备）
- **使用场景**（见面洽谈/内容创作/投资评估）

如果用户同时给出人名+机构，转交 `entity-research` skill 处理人物背调部分，本skill聚焦领域知识层。

### Step 2：extended search调研

启动 `launch_extended_search_task`，调研命令模板：

```
请对【X领域】进行深度调研，目标读者是一位早期AI孵化投资人（非X领域专业背景），
需要为与该领域顶级专家的洽谈做准备。

调研维度一：底层概念体系
- X领域的核心概念、定义、基本逻辑链
- 关键术语及其实际含义（避免堆砌术语，要解释清楚）
- 涉及的主要主体/角色及其商业模式

调研维度二：行业现状与关键数据
- 中国市场的规模、增速、政策背景
- 主要痛点和结构性矛盾
- 近1-2年最重要的政策/市场变化

调研维度三：AI与该领域的交叉点
- AI技术正在如何改变这个领域
- 有哪些投资机会或值得关注的创业方向

调研维度四：与【具体机构/人物】相关的背景（如有）
- 机构的定位、商业模式、核心竞争力
- 人物的职业路径和核心能力

请用中文回答，要有深度，有具体数据，避免表面化总结。
```

### Step 3：输出领域认知地图

调研完成后，整理为结构化Markdown，包含：

```
一、底层概念体系（投资人视角的解释，不是教科书定义）
二、行业现状与关键数据（有数据，有判断）
三、AI交叉机会（和Faye的投资视角相关的部分）
四、机构/人物背景（如有）
五、洽谈准备建议（基于以上，给出3-5个高价值问题）
```

**输出标准**：不是堆砌信息，每个部分要有一句「这意味着什么」的判断。

## PHASE 2：录音稿提炼 → 完整文档

### 触发条件

用户上传 `.docx` 格式的录音转写文稿，并指定：
- 聚焦哪个说话人（例：「说话人3」「赵永辉」）
- 或聚焦哪类内容（例：「知识产权相关的所有专业知识」「合作意向相关的」）

### Step 1：读取录音稿

```bash
pandoc /mnt/user-data/uploads/[文件名].docx -o /tmp/transcript.md
```

识别说话人标记格式（常见格式：`说话人 X HH:MM:SS`），提取指定说话人的所有发言。

提取命令参考：
```python
# 按说话人切割，提取指定说话人的全部内容
# 注意：每个发言段=说话人标记行 + 后续内容行（到下一个说话人标记前）
```

### Step 2：专业内容分类提炼

读取指定说话人的所有发言后，按以下维度进行**主题聚类**（不是按时间顺序堆砌）：

**A. 专业知识层**（这个人在说什么领域知识）
- 核心概念的一手解释
- 行业判断和数据（注意：这是从业者视角，可能比公开数据更真实）
- 反常识的洞察（最有价值的部分）

**B. 机构运作层**（这个人/机构是怎么运转的）
- 商业模式的具体描述
- 合作逻辑和价值主张
- 真实案例（最能揭示运作逻辑的部分）

**C. 市场判断层**（这个人对行业的主观判断）
- 他认为什么有价值/没价值
- 他对竞争格局的看法
- 他对未来趋势的预判

**D. 合作信号层**（这个人表达了哪些合作意向）
- 明确说了的
- 隐含在措辞里的
- 他主动提出的条件

### Step 3：与Phase 1背调融合

将从录音稿提炼的内容，与Phase 1的background research合并：

```
融合原则：
- Phase 1提供的是「行业通用认知」（第三方视角）
- Phase 2提炼的是「这个人的一手判断」（从业者视角）
- 两者形成「客观背景 + 主观洞察」的双层结构
- 有冲突时：标注出来，不要强行统一
- 有补充时：Phase 2的内容通常更具体，优先级更高
```

### Step 4：输出完整文档（.docx）

**调用docx skill**生成格式规范的Word文档。详见 `references/output-format.md`

文档结构模板：
```
封面：文档标题 + 说明（数据来源、日期、使用场景）

第一部分：[领域名]核心概念体系
  - 从底层逻辑开始，不是从术语开始
  - 每个核心概念：定义 + 投资视角的意义 + 关键数据

第二部分：行业现状与结构性特征
  - 规模数据
  - 核心矛盾/痛点
  - 政策背景
  - AI交叉机会

第三部分：[机构名] + [人名]深度背调
  - 机构：定位、商业模式、核心资产、竞争优势
  - 人物：职业路径、核心能力、真实判断（引用原话）
  - 典型案例（运作逻辑最清晰的那个）

第四部分：合作路径分析
  - 双方价值互补矩阵
  - 可落地的合作形式
  - 合作意愿信号（来自录音）

附录：关键概念速查表
```

## 核心操作规范

### 引用原话的使用原则

录音中的原话非常宝贵，因为它是对方**真实判断**的直接呈现。使用规则：

- **必须引用**：反常识的判断、具体数据、典型案例、合作意愿表达
- **引用格式**：用callout box或blockquote独立呈现，不要混在叙述里
- **引用后**：加一段分析，说明这句话背后的逻辑意义（不只是复述）
- **不引用**：闲聊、寒暄、重复性内容、无实质信息的过渡语

### 去除特定身份视角

最终文档是**客观第三方视角**，不是某一方的调研报告：
- 不写「作为投资人」「从我们的角度」
- 改写为：「对于关注早期技术转化的机构而言...」「对接方需要具备...」
- 合作分析部分用「双方」表述，不预设哪方是主动方

### 什么内容不提炼

- 说话人在录音中的闲谈（和专业内容无关的）
- 无法单独成立的上下文依赖内容（离开了会谈语境就不成立的）
- 个人隐私或敏感商业信息（如具体融资数字、人事变动等）
