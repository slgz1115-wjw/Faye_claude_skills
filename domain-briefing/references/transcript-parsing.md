# 录音稿解析参考

## 常见录音转写格式

### 格式一：标准说话人标记（最常见）
```
说话人 1 00:00:05\
[发言内容]

说话人 2 00:00:23\
[发言内容]

说话人 1 00:00:45\
[发言内容]
```

### 格式二：带姓名的标记
```
张总 00:15:32
[发言内容]

主持人 00:16:01
[发言内容]
```

### 格式三：纯时间轴
```
[00:05:12] [发言内容]
[00:05:45] [发言内容]
```

## 标准提取流程

### Step 1：读取文件
```bash
pandoc /mnt/user-data/uploads/[文件名].docx -o /tmp/transcript.md
wc -l /tmp/transcript.md  # 确认文件大小
```

### Step 2：识别说话人格式
```bash
# 检查说话人标记格式
grep -n "说话人\|Speaker\|主持人" /tmp/transcript.md | head -20
```

### Step 3：提取指定说话人内容
```python
# 标准提取脚本
with open('/tmp/transcript.md', 'r') as f:
    lines = f.readlines()

target_speaker = '说话人 3'  # 根据用户指令修改
segments = []
i = 0

while i < len(lines):
    line = lines[i]
    if target_speaker in line:
        segment = [line.strip()]
        j = i + 1
        while j < len(lines):
            next_line = lines[j].strip()
            # 遇到任何说话人标记就停止
            if '说话人 ' in next_line and any(t in next_line for t in ['00:', '01:', '02:']):
                break
            if next_line:
                segment.append(next_line)
            j += 1
        segments.append('\n'.join(segment))
        i = j
    else:
        i += 1

full_text = '\n\n---\n\n'.join(segments)
with open('/tmp/speaker_target.txt', 'w') as f:
    f.write(full_text)

print(f"提取段落数: {len(segments)}")
print(f"总字符数: {len(full_text)}")
```

### Step 4：内容质量筛查

提取完成后，先做一次快速扫描，过滤掉：
- 纯闲聊（「好的」「嗯嗯」「是的」「稍等」等无实质内容）
- 打断性回应（只有1-2个字的回应）
- 重复性内容（同一意思说了多次，保留最完整的那次）

**判断标准**：这段话如果放在文档里，对不了解这个人的读者有信息增量吗？有就保留，没有就过滤。

## 内容聚类方法

提取完成后，不要按时间顺序整理，要按**主题**聚类：

### 聚类维度参考

**专业知识类**（该人在解释/分析什么）
- 概念解释
- 行业判断
- 数据引用
- 反常识洞察

**机构介绍类**（该人在介绍自己的机构/业务）
- 公司背景
- 服务内容
- 客户案例
- 规模数据

**合作意向类**（该人在表达合作想法）
- 明确提出的合作形式
- 隐含的合作期待
- 资源置换逻辑

**价值判断类**（该人在表达自己的观点/偏好）
- 他认为什么重要
- 他不认可什么
- 他对竞争对手的看法

## 高价值内容识别标准

以下类型的发言要**重点标记**，这些是文档中最有价值的内容：

### 🔴 最高价值：反常识判断
识别特征：「大家都说X，但其实Y」「你可能以为是X，实际上是Y」
例：「之前是有个数据是3.6%（产业化率），然后全球的话是40%……现在专利第一是量要降。」

### 🔴 最高价值：具体案例（有数字有结果）
识别特征：具体公司名 + 具体操作 + 具体结果
例：「我们是以服务换股权的方式……这个案例后来大概翻了十几倍，一个亿。」

### 🟡 高价值：一手行业数据
识别特征：具体数字 + 行业背景
例：「华进在全国有1500人，在这边有150多人」「我们一年做到8000万」

### 🟡 高价值：运作逻辑揭示
识别特征：「我们的思路是」「我们的模式是」「这样做的原因是」

### 🟢 参考价值：主观判断
识别特征：「我觉得」「在我看来」「我们比较关注」
→ 这类内容揭示该人的价值观，对判断合作契合度有价值

## 常见陷阱

### 陷阱一：把上下文依赖的话孤立提炼
❌ 错误：录音说「这个项目翻了十几倍」→ 文档只写「翻了十几倍」
✅ 正确：解释是哪个项目、什么时间、什么操作，然后写「翻了十几倍」

### 陷阱二：把口语直接搬入文档
❌ 错误：「这东西以后肯定有长远前景，但我就需要一笔钱然后需要一帮人」
✅ 正确：提炼为「技术型创始人普遍面临的资金+团队双缺口，是成果转化的典型瓶颈」并附原话作为例证

### 陷阱三：忽略否定性内容
说话人说「我们不做X」「我们对Y比较保留」「我们不太擅长Z」——
这些**否定性内容**非常重要，揭示了机构的边界和局限，务必纳入分析。

### 陷阱四：把合作意向过度解读
「可以可以」「没问题」「可以考虑」在中文商务对话中是礼貌性回应，不等于强合作意愿。
真正的合作信号是：主动提出具体形式、主动报价、主动提出下一步行动。
