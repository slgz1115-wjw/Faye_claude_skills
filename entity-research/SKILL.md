# Entity Research Skill v2 — Faye专用调研与会谈管理系统

## 核心原则

这个框架不是用来整理信息的，是用来**形成判断和行动指令**的。每次调研最终必须回答三个问题：
1. 对方真正想要什么？
2. 我能给他什么他没有的？
3. 这次见面，我的最优策略是什么？

信息分三类标注：**已知**（多源验证）/ **推断**（有依据）/ **待验证**（需见面确认）

---

## PHASE 0：解析输入格式

收到调研请求时，先识别格式：

```
【机构名-人名】→ 主调研对象 = 机构，人作为附属分析
【人名-机构名】→ 主调研对象 = 人，机构作为背景分析
```

两种情况都同时做人和机构的分析，只是报告的侧重和深度不同。

调研目的如果用户未说明，默认推断为"初次见面准备"。

## PHASE 1：调研报告

### Step 1：信息采集优先级

**个人信息源**

| 优先级 | 来源 | 看什么 |
|--------|------|--------|
| 🔴 核心 | LinkedIn/领英 | 职业时间线、跳槽逻辑、技能背书 |
| 🔴 核心 | 天眼查/企查查 | 是否持有其他公司股权/法人/高管 |
| 🟡 重要 | 微博/Twitter | 真实关注点、顺逆境发言、互动圈子 |
| 🟡 重要 | 公众号/知乎/即刻 | 深度思考质量、行业判断 |
| 🟡 重要 | 百度/Google | 姓名+公司 → 新闻报道、演讲实录、访谈 |
| 🟢 补充 | Crunchbase/IT桔子 | 投资记录、相关公司 |
| 🟢 补充 | GitHub/知网 | 技术或学术背景专项 |

**机构信息源**

| 优先级 | 来源 | 看什么 |
|--------|------|--------|
| 🔴 核心 | 天眼查/企查查/启信宝 | 股权结构、实际控制人、法律风险 |
| 🔴 核心 | 官网 | 关于我们、团队、投资组合、新闻 |
| 🟡 重要 | IT桔子/Crunchbase | 融资历史、投资组合、合投网络 |
| 🟡 重要 | 招聘网站 | 在招职位 → 业务扩张信号 |
| 🟢 补充 | 36氪/虎嗅/投中 | 报道、专访 |
| 🟢 补充 | 脉脉 | 员工口碑（需甄别） |

**采集原则**：区分"对方希望你看到的样子"vs"真实样子"。缺失的信息本身就是信号。PR能力强的主体，正面形象要打折。

---

### Step 2：四维分析（人版 / 机构版同时运行）

详细分析框架见 `references/four-dimensions.md`

四维速查：

| 维度 | 核心问题 | 最大风险 |
|------|----------|----------|
| **本体** | 它是什么？核心资产是否真实可验证？ | 粉饰，看有什么更要看缺什么 |
| **动力** | 为何而动？表层vs深层驱动力是否一致？ | 激励扭曲，个人和组织目标不对齐 |
| **连接** | 处于什么生态位？连接了谁/没连接谁？ | 孤立或过度依赖单一连接 |
| **演化** | 从哪来、现在哪、要去哪？ | 路径依赖，被过去成功绑架 |

---

### Step 3：会谈策略输出（**必须包含，这是最终交付**）

报告的最后一节必须包含以下全部内容：

#### A. 对方想从这次见面得到什么？
基于动力维度推断，列出1-3个核心诉求。

#### B. 我能给他什么他没有的？
基于连接维度分析，从Faye的身份出发，精确说出差异化价值：
- ZJU X-Lab的项目流（有什么类型的项目，数量级，阶段）
- Delta X的孵化资源（蚂蚁生态、PMF方法论、浙大人脉）
- Faye本人（投资判断、网络节点、媒体曝光能力）

#### C. 这次见面，我应该做什么、说什么、不说什么？

**可以说**（能建立信任的话）：
- 具体项目案例（不透露商业细节，但展示判断力）
- 对对方所在领域的真实判断（哪怕有点尖锐）
- Faye自己遇到的问题，寻求对方的专业意见

**不该说**（避免的话术）：
- 过早抛出"合作"二字（显得功利）
- 泛泛介绍Delta X的品牌（对方没兴趣）
- 说对方"也许"能帮到我们（弱化自己价值）

**不能说**（坚决避免）：
- 披露在谈项目的融资细节
- 暗示浙大资源是独占的（会引起保护意识）

#### D. 如何让自己在对方眼里变得可信？

从以下角度选择最合适的建立信任方式：
- **知识信任**：对对方领域展示真实理解（用具体例子，不泛泛）
- **网络信任**：提到共同认识的人/机构（需真实，不能硬造）
- **价值先行**：直接给出对方有用的信息/连接（不求回报的那种）
- **身份信任**：浙大X-Lab + 蚂蚁背景的具体含义解释（不要假设对方了解）

#### E. 3-5个开场问题（按对话目标排序）

每个问题标注：
- 问题本身
- 背后想验证的假设
- 如果对方这样回答，说明什么

## PHASE 2：会后复盘

用户发来会谈记录（文字/截图/语音转文字）时触发。

输出一份**会后档案**，结构如下：

### 2.1 关键信息提取
从对话中提炼出：
- 对方透露的真实诉求（不是说了什么，是暗示了什么）
- 对方的决策权判断（他能决定什么？需要向谁汇报？）
- 对方对Delta X / Faye的第一印象（从反应推断）
- 意外收获的信息（没预期到但很有价值的）

### 2.2 潜在合作机会
对每个潜在合作，标注：
- 合作形式（具体到"做什么"层面）
- 合作意愿度（高/中/低，基于对话信号判断）
- 推进障碍（什么可能阻碍它落地）
- 下一步行动（谁、做什么、什么时间节点）

### 2.3 话术得失分析
结合调研阶段的会谈策略，逐条对比：
- ✅ 哪些话术奏效了（对方反应积极，信息获取顺利）
- ❌ 哪些话术没奏效（对方回避、话题转移、明显冷场）
- ⚠️ 哪些可以说得更好（有更优的切入方式）
- 🔍 哪些调研假设被验证/被推翻了

### 2.4 关系状态评估
见面之后，双方关系在哪个阶段：
- 陌生人 → 认识了 → 有好感 → 有共同利益 → 深度合作

当前在哪里，下次互动的目标是推进到哪里。

### 2.5 下次跟进策略
- 跟进时机（什么时候、以什么理由）
- 跟进内容（给什么，问什么）
- 需要在下次见面前做的准备

---

## PHASE 3：档案持续更新

每次用户说"补充信息"、"更新一下"、"他说了XXX"时触发。

在既有档案基础上：
1. 标注信息来源（见面/朋友反馈/公开信息）
2. 更新四维分析中受影响的部分
3. 重新评估合作机会和关系状态
4. 更新下次跟进策略

**档案是活的，不是一次性的。**
